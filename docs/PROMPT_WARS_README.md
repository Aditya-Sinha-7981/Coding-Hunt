# Prompt Wars — Technical README

## What We Are Building

A live event web app for a college fest. Participants register, submit
AI-generated images each round, and vote in live majority/minority polls.
Points are awarded automatically. A live leaderboard runs on the projector.

No traditional server. Google Sheets is the database. Google Apps Script
is the backend API. A static HTML/CSS/JS site hosted on Netlify is the
frontend. The host controls everything from one Control tab in Sheets.

---

## System Overview

```
[Participant Website]  <-->  [Apps Script API]  <-->  [Google Sheets]
      |                                                      |
      | register / submit image / vote                       |
      |                                                      |
[Admin Panel]  -------> Control Tab (open/close rounds)      |
                                                             |
[Gemini API] <--- triggered by Apps Script on new submission-+
```

## Google Sheets Structure

One spreadsheet. Eight tabs. Do not rename tabs — names are hardcoded.

---

### Tab 1: Teams

Stores all registered participants.

| Col | Header        | Notes                                        |
|-----|---------------|----------------------------------------------|
| A   | Team ID       | Auto-generated: T001, T002...                |
| B   | Team Name     | Entered at registration                      |
| C   | Member 1      | Required                                     |
| D   | Member 2      | Optional                                     |
| E   | Member 3      | Optional                                     |
| F   | Email         | Used as unique login key                     |
| G   | Auth Token    | Random string generated at registration      |
| H   | Registered At | Timestamp, auto-filled                       |

Rules:
- Email must be unique. Registration rejected if email already exists.
- Team ID is assigned by Apps Script on registration, never by participant.
- Auth Token is a random 16-character alphanumeric string generated by
  Apps Script on registration. Stored here and returned to client once.
  Every subsequent API call must send teamId + authToken. Backend
  validates both before processing any write action.
- localStorage stores teamId, authToken, and email on client side.
  On page load these are read back to restore session state.

---

### Tab 2: Rounds

One row per round. Pre-filled by host before event starts.

| Col | Header        | Notes                                      |
|-----|---------------|--------------------------------------------|
| A   | Round Number  | 1, 2, 3, 4, 5...                           |
| B   | Round Type    | image OR poll                              |
| C   | Reference URL | For image rounds: target image URL         |
| D   | Prompt Text   | Written prompt shown to participants       |
| E   | Poll Question | For poll rounds only                       |
| F   | Option A      | For poll rounds only                       |
| G   | Option B      | For poll rounds only                       |
| H   | Notes         | Internal notes for host, not shown to users|

Rules:
- Fill this entire tab before the event starts.
- Poll Mode lives only in Control tab, not here. Single source of truth.
- Reference URL for image rounds must be a publicly accessible image link.

---

### Tab 3: Control

The host's live control panel. Only one row of data (row 2).
Host edits this tab during the event to manage round state.

| Col | Header        | Value Options                               |
|-----|---------------|---------------------------------------------|
| A   | Round Number  | Any number matching Rounds tab              |
| B   | Round Type    | image / poll (auto-filled from Rounds tab)  |
| C   | Status        | open / closed                               |
| D   | Poll Mode     | majority / minority / secret                |
| E   | Poll Reveal   | majority / minority (for secret rounds only)|

Rules:
- Status = open means website accepts submissions and votes.
- Status = closed means ALL submissions and votes are rejected at the
  backend level regardless of what the frontend shows. Enforced
  server-side, not just by disabling buttons.
- Changing Round Number auto-fetches Round Type from Rounds tab.
- Poll Mode is the single source of truth. Rounds tab has no Poll Mode.
- Poll Reveal is only set when host reveals a secret round result.
- This is the only tab the host touches during the live event.

---

### Tab 4: Submissions

Image submission queue. One row per submission.

| Col | Header       | Notes                                    |
|-----|--------------|------------------------------------------|
| A   | Submission ID| Auto-generated: S001, S002...            |
| B   | Team ID      | From Teams tab                           |
| C   | Round Number | From Control tab at time of submission   |
| D   | Image URL    | Google Drive public URL of uploaded image|
| E   | Submitted At | Timestamp                                |
| F   | Score        | Blank until Gemini scores it             |
| G   | Scored At    | Timestamp, filled after scoring          |
| H   | Status       | pending / scored / error                 |

Rules:
- Only one submission per team per round. Enforced atomically:
  LockService acquires lock → check for existing row → insert → release.
  Second attempt returns alreadySubmitted = true.
- Backend checks Control tab status = open before accepting any submission.
  If status = closed, returns roundClosed error immediately.
- Score filled by Apps Script after Gemini evaluation.
- Status = error means Gemini call failed. Team gets participation points.
  Host can manually trigger rescore from admin panel.

---

### Tab 5: Poll Votes

One row per vote action. Multiple rows per team per poll (tracks switches).

| Col | Header       | Notes                                         |
|-----|--------------|-----------------------------------------------|
| A   | Vote ID      | Auto-generated                                |
| B   | Team ID      | From Teams tab                                |
| C   | Round Number | Current round from Control                    |
| D   | Vote         | A or B                                        |
| E   | Timestamp    | Every switch logged separately                |
| F   | Is Final     | FALSE for all, TRUE set at round close        |

Rules:
- Every switch is logged as a new row. Full history kept.
- Is Final = TRUE set by Apps Script in one batch pass when host
  closes the round. Only these rows used for scoring.
- Backend enforces 5 second cooldown by checking timestamp of most
  recent row for that team in that round. Client-side cooldown is
  additional UX only, not the enforcement layer.
- Backend rejects votes if Control status = closed.
- LockService wraps every vote write.

---

### Tab 6: Scores

Final points per team per round. Source of truth for leaderboard.

| Col | Header       | Notes                                    |
|-----|--------------|------------------------------------------|
| A   | Team ID      | From Teams tab                           |
| B   | Round Number | Which round                              |
| C   | Round Type   | image or poll                            |
| D   | Points       | Final points awarded                     |
| E   | Detail       | Score% for image, vote+mode for poll     |

Rules:
- One row per team per round.
- Written by Apps Script after scoring completes.
- Host can directly edit Points column for manual correction.
  Leaderboard rebuilds automatically via onEdit trigger.

---

### Tab 7: Leaderboard

Auto-rebuilt every time Scores tab changes.
This is what the projector page reads.

| Col | Header      | Notes                    |
|-----|-------------|--------------------------|
| A   | Rank        | 1, 2, 3...               |
| B   | Team Name   | From Teams tab           |
| C   | Total Points| Sum from Scores tab      |
| D   | Last Updated| Timestamp of last rebuild|

Rules:
- Sorted descending by Total Points.
- Rebuilt fully on every score write.
- Password protected view on website.

## Points System

### Image Rounds
- 80-100  →  50 points
- 60-79   →  35 points
- 40-59   →  20 points
- Below 40 →  5 points
- No submission → 0 points
- Gemini error → 5 points (participation, not penalised for API failure)

### Poll Rounds
- Majority round: winning side gets 30 points.
- Minority round: winning side gets 50 points.
- Secret round: winning side gets 40 points.
- No vote → 0 points.

## Authentication

Every write action requires teamId + authToken. Backend validates both.

Flow:
1. Register → Apps Script generates random 16-char alphanumeric token.
2. Token stored in Teams tab col G.
3. Token returned to client once in registration response.
4. Client stores teamId + authToken + email in localStorage.
5. Every POST includes both fields.
6. Backend looks up teamId row, checks authToken matches, rejects if not.

If a team loses localStorage (different device, cleared browser):
- They can re-enter email to confirm registration exists.
- They cannot recover authToken. Cannot submit for that round.
- Announce before event: use one device, do not clear browser.

## LockService Pattern

Every function that writes to Sheets uses this exact pattern.
No exceptions.

```javascript
var lock = LockService.getScriptLock();
lock.waitLock(10000);
try {
  // 1. check (read existing data)
  // 2. write (append or update)
} finally {
  lock.releaseLock();
}
```

Functions requiring LockService:
- register
- submitImage
- submitVote
- scoreSubmissions (Gemini trigger)
- updateLeaderboard

If lock cannot be acquired in 10 seconds, return error to client.
Client shows "please try again" message.

## Apps Script API

Deployed as Web App. One URL. GET for reads, POST for writes.

### GET Endpoints

```
?action=status
Current round number, type, status. Cached 3 sec TTL.

?action=leaderboard&password=XXXX
Ranked team list. Rejected if wrong password.

?action=roundinfo&round=3
Full row from Rounds tab for that round number.

?action=pollresults&round=3
Live vote counts for A and B. Public, no auth needed.
Used for the live bar chart.
```

### POST Endpoints

```json
{ "action": "register",
  "teamName": "", "member1": "", "member2": "", "member3": "",
  "email": "" }
Returns: { success, teamId, authToken, error }
authToken returned here only. Never returned again.

{ "action": "checkEmail", "email": "" }
Returns: { success, exists, teamId }
Does NOT return authToken.

{ "action": "submitImage",
  "teamId": "", "authToken": "", "round": 1,
  "imageData": "base64string" }
Returns: { success, submissionId, alreadySubmitted, roundClosed, error }

{ "action": "submitVote",
  "teamId": "", "authToken": "", "round": 1,
  "vote": "A", "timestamp": 1234567890 }
Returns: { success, cooldownRemaining, roundClosed, error }

{ "action": "setControl",
  "adminPassword": "", "round": 1, "status": "open",
  "pollMode": "majority" }
Returns: { success, error }

{ "action": "revealPoll",
  "adminPassword": "", "round": 3, "mode": "majority" }
Returns: { success, error }
Sets Poll Reveal in Control. Triggers poll scoring immediately.

{ "action": "rescoreRound",
  "adminPassword": "", "round": 1 }
Returns: { success, requeued, error }
Resets error-status submissions to pending so trigger rescores them.
```

## Image Upload Flow

Participant selects image on phone.
Frontend compresses before upload. Backend saves to Drive.

Frontend compression (in participant.html):
```javascript
function compressImage(file, callback) {
  var reader = new FileReader();
  reader.onload = function(e) {
    var img = new Image();
    img.onload = function() {
      var canvas = document.createElement('canvas');
      var max = 800;
      var w = img.width, h = img.height;
      if (w > h && w > max) { h = h * max / w; w = max; }
      else if (h > max) { w = w * max / h; h = max; }
      canvas.width = w; canvas.height = h;
      canvas.getContext('2d').drawImage(img, 0, 0, w, h);
      callback(canvas.toDataURL('image/jpeg', 0.7).split(',')[1]);
    };
    img.src = e.target.result;
  };
  reader.readAsDataURL(file);
}
```

Target: under 400KB per image after compression.
Apps Script receives base64, decodes, saves to Drive via DriveApp,
makes file publicly readable, returns URL, stores URL in Submissions tab.

Drive folder structure:
```
Prompt Wars Event/
  Round 1/
    T001_S001.jpg
  Round 2/
    ...
```

## Gemini Scoring

Time-based trigger every 60 seconds.

Flow:
1. Acquire LockService lock.
2. Fetch all Submissions rows where Status = pending.
3. For each row:
   - Call Gemini Vision with submitted URL + reference URL.
   - Prompt: "Compare these two images. Return ONLY a raw JSON object
     with one key called score containing an integer 0 to 100 for
     visual and thematic similarity. No markdown, no explanation."
   - Parse response safely:
     ```javascript
     var raw = response.getContentText().trim()
                       .replace(/```json|```/g, '').trim();
     try {
       var parsed = JSON.parse(raw);
       var score = parseInt(parsed.score);
       if (isNaN(score) || score < 0 || score > 100) throw new Error();
     } catch(e) {
       // retry once after 2 seconds
       // if retry fails: score = 5, status = error
     }
     ```
4. Write points to Scores tab.
5. Update Submission status to scored or error.
6. After all rows processed, rebuild Leaderboard once.
7. Release lock.

## Website Structure

Four pages. Static. Hosted on Netlify.

```
/index.html        — Registration
/participant.html  — Round view (image or poll)
/admin.html        — Admin control panel
/leaderboard.html  — Projector leaderboard
```

### index.html
- Fields: team name, member 1 (required), members 2-3 (optional), email.
- Submit button disables on click. Re-enables only on error.
- On success: store teamId, authToken, email in localStorage.
  Redirect to participant.html.
- On load: if localStorage has teamId + authToken, skip to participant.html.

### participant.html
- On load: read teamId + authToken from localStorage.
  If missing, redirect to index.html.
- Poll status endpoint every 5 seconds.
- Image round:
  - Show prompt and reference image.
  - File input → compress → submit.
  - Button disables on click until response received.
  - On success: store round in submittedRounds in localStorage.
    Show confirmation. Hide form.
  - On load: if round already in submittedRounds, show confirmation
    without calling API.
- Poll round:
  - Show question, Option A and Option B buttons.
  - Live bar chart from pollresults endpoint, refreshed every 3 seconds.
  - Chart shows data from 3 seconds ago (artificial delay).
    Label: "updates every few seconds".
  - After voting, 5 second cooldown. Button shows countdown.
    Cooldown enforced client-side for UX and server-side for integrity.
  - Store current vote per round in localStorage. Restored on refresh.
- Status = closed: show waiting screen with last result if available.

### admin.html
- Password prompt on load. Stored in sessionStorage.
- Live view of Control tab, refreshed every 5 seconds.
- Controls: round number input, status toggle, poll mode dropdown,
  reveal poll button (visible only when mode = secret),
  manual rescore button.
- Live count: pending / scored / error submissions for current round.

### leaderboard.html
- Password prompt on load.
- Fetches leaderboard every 10 seconds.
- Full screen, large text, high contrast for projector.
- Shows current round and status at top.

## localStorage State Map

```
teamId          — set at registration
authToken       — set at registration, never changes
email           — set at registration
submittedRounds — array: [1, 2] means submitted rounds 1 and 2
votedRounds     — object: { "3": "A" } means voted A in round 3
lastResult      — last score or poll result shown to user
```

On every page load, participant.html reads all keys and renders
correct state without redundant API calls.

## Known Limits

- Apps Script execution: 6 min cap per run.
  50 images at 3 sec each = 150 sec. Safe.
- Daily quota: 90 min total. 3 hour event well within this.
- Drive storage: 50 teams x 7 rounds x 400KB = ~140MB. Fine.
- LockService timeout: 10 sec. Extremely rare to hit.
- CacheService TTL: 3 sec on status. Round changes propagate fast.

## Event Day Checklist

Before:
- Rounds tab fully filled (prompts, reference URLs)
- Config tab set (password, points, Gemini key, max teams)
- Control tab: Round 1, status = closed
- Test registration on a real phone
- Test one image round end to end including Gemini score appearing
- Test one poll round including close and scoring
- Leaderboard on projector, password confirmed working
- QR code printed or displayed for participant URL
- Backup Google Form open on host phone

During:
- Image round: set round number, confirm reference URL in Rounds tab,
  flip status open, announce, flip closed when done,
  wait 60-90 sec for scores, show leaderboard.
- Poll round: set round number, set poll mode, flip open,
  watch votes, flip closed, scores instant, show leaderboard.
- Secret round: flip closed, set Poll Reveal, click Reveal in admin.

Emergency:
- Gemini fails: click Rescore in admin. If still fails, edit Scores tab
  directly. Leaderboard rebuilds on save.
- Student lost session: cannot recover authToken. They sit out that round.
  Announce device policy before event starts.
- Connection drop: student reloads. localStorage restores state.
